JARFIS 고급 패턴 완전 정복 — 미팅과 작업 항목 연결부터 에이전트 역할 커스터마이징, Gate 관리, 중단 복구까지.

## Meeting-to-Work 연결

JARFIS는 미팅 결과를 개발 워크플로우에 직접 연결합니다. 먼저 미팅을 진행해 구조화된 출력물을 만들고, 그 출력물을 work 명령어에 연결하면 파이프라인 전체가 완전한 컨텍스트를 갖고 시작됩니다.

<FlowDiagram nodes={[
  { label: "/jarfis:meeting", variant: "meeting" },
  { label: "summary.md + decisions.md + meeting-notes.md" },
  { label: "/jarfis:work --meeting", variant: "work" },
  { label: "Phase 0에서 미팅 컨텍스트 로드" }
]} />

### 단계별 진행

1. 주제와 함께 미팅 명령어를 실행합니다:

```
/jarfis:meeting Define the authentication strategy for the mobile app
```

실행 결과로 `summary.md`, `decisions.md`, `meeting-notes.md`가 생성됩니다.

2. `--meeting` 플래그로 미팅 출력물을 work 명령어에 연결합니다:

```
/jarfis:work Implement mobile auth with biometric login --meeting auth-strategy-meeting
```

3. Phase 0 (Pre-flight)이 자동으로 미팅 컨텍스트를 로드합니다. 이후 모든 Phase의 에이전트가 미팅에서 결정된 사항을 참조합니다.

<Callout>
  **팁:** `--meeting` 플래그는 미팅 출력물 디렉토리를 참조합니다. Phase 0이 모든 미팅 산출물을 로드하므로 파이프라인의 모든 에이전트가 논의된 결정과 제약 사항에 접근할 수 있습니다.
</Callout>

## Role 설정

JARFIS는 `.jarfis-state.json`의 `required_roles` 필드를 사용해 워크플로우에서 활성화할 에이전트 역할을 결정합니다. 이 설정이 어느 Phase를 실행할지, Phase 4에서 병렬 작업을 어떻게 분배할지를 제어합니다.

```json
{
  "required_roles": {
    "backend": true,
    "frontend": true,
    "ux": false,
    "devops": false,
    "security": true
  }
}
```

</Callout>

## 완료된 워크플로우 이어서 진행

`/jarfis:continue`를 사용하면 완료된 워크플로우를 기반으로 작업을 이어갈 수 있습니다 — 테스트 추가, 엣지 케이스 수정, 범위 확장 등 처음부터 다시 시작하지 않아도 됩니다.

```
/jarfis:continue Add unit tests for the auth module
/jarfis:continue Fix the token refresh edge case --mode fix
```

<ModeGrid modes={[

    flag: "--mode extend"
  }
]} columns={2} />

`--workflow path`로 특정 이전 워크플로우의 출력물을 타겟으로 지정해 이어서 진행할 수도 있습니다.

## 프로젝트 프로파일

프로젝트 프로파일은 JARFIS가 코드베이스 전체를 파악할 수 있게 합니다 — 저장소 구조, 의존성, 빌드 도구, 컨벤션까지 — 모든 워크플로우가 정확한 프로젝트 컨텍스트와 함께 시작됩니다.

<RecoveryFlow steps={[
  { text: <><strong><code>/jarfis:project-init</code></strong> — 저장소 구조, 의존성, 빌드 도구, 컨벤션을 분석해 <code>.jarfis/project-profile.md</code>를 생성합니다.</> },
  { text: <><strong><code>/jarfis:project-update</code></strong> — 프로젝트가 발전했을 때 수동으로 추가한 컨텍스트를 유지하면서 프로파일을 갱신합니다.</> }
]} />

<Callout>
  **팁:** 저장소에 JARFIS를 처음 설정할 때 `/jarfis:project-init`을 한 번 실행하고, 주요 의존성이나 구조 변경 후에는 `/jarfis:project-update`를 사용하세요.
</Callout>

## 컨텍스트 압축 후 워크플로우 복구

JARFIS는 `.jarfis-state.json`을 통해 Phase와 진행 상태를 디스크에 저장합니다. 별도의 재개 명령어가 필요 없습니다 — 시스템이 중단된 워크플로우를 자동으로 감지하고 재개합니다.

<RecoveryFlow steps={[
  { text: <><strong>세션 종료</strong> — 연결 끊김, 타임아웃, 또는 컨텍스트 압축</> },
  { text: <><strong><code>/jarfis:work</code> 재실행</strong> — 이전과 동일한 작업 설명 사용</> },
  { text: <><strong>JARFIS가 기존 상태 파일 감지</strong> — 진행 중인 워크플로우가 있는 <code>.jarfis-state.json</code> 발견</> },
  { text: <><strong>마지막 체크포인트에서 재개</strong> — 이전 세션이 중단된 정확한 지점부터 작업 재개</> }
]} />

<Callout>
  **핵심:** 별도의 재개 명령어는 없습니다. 동일한 설명으로 `/jarfis:work`를 실행하기만 하면 JARFIS가 저장된 상태 파일을 통해 자동으로 복구를 처리합니다.
</Callout>

## JARFIS 자체 수정

JARFIS 자체를 수정하거나 확장하고 싶을 때 — 새 명령어 추가, 에이전트 동작 변경, 파이프라인 구조 업데이트 — `/jarfis:implement`를 사용하세요.

```
/jarfis:implement Add a /jarfis:standup command for daily status reports
```

<Callout>
  **작동 방식:** `/jarfis:implement`는 JARFIS 시스템 구조를 이해하고 프롬프트 템플릿, 명령어 정의, Phase 핸들러, 문서 등 영향받는 모든 파일에 변경 사항을 적용합니다.
</Callout>
