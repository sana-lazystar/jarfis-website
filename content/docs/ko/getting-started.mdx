# Quick Start

AI와 함께 소프트웨어를 만드세요 — 슬래시 명령어 하나, 11명의 전문 Agent, Human Gate가 있는 구조화된 Phase.

## 사전 요구사항

시작하기 전에 다음을 준비하세요:

- **Claude Code** (Anthropic) — JARFIS는 Claude Code 안에서 슬래시 명령어 워크플로우로 동작합니다
- API 접근 권한이 있는 Claude 계정
- **Git** >= 2.x (Phase 0의 브랜치 관리를 위해)
- [JARFIS GitHub 저장소](https://github.com/sana-lazystar/jarfis) 클론 또는 접근 가능 상태

## 설치

JARFIS는 GitHub 저장소의 `install.sh` 스크립트로 설치합니다. 터미널에서 다음을 실행하세요:

```bash
bash install.sh
```

이 명령어는 Claude Code 환경에 JARFIS 슬래시 명령어를 등록합니다. 설치 후 다음 프로젝트 구조를 사용할 수 있습니다:

```
your-project/
├── .jarfis/
│   ├── context.md             # 프로젝트별 컨텍스트
│   └── project-profile.md     # 프로젝트 프로파일
├── .jarfis-state.json         # 워크플로우 상태 추적
└── ~/.claude/
    └── jarfis-learnings.md    # 전역 학습 파일 (프로젝트 간 공유)
```

> **참고**: `/jarfis:install` 명령어는 없습니다. 설치는 반드시 `bash install.sh`로만 진행합니다.

## 첫 워크플로우 실행

프로젝트 디렉토리에서 Claude Code를 열고 워크플로우를 시작하세요:

```
/jarfis:work Build a user authentication system with JWT tokens
```

JARFIS가 요청을 분류(Phase T — Triage)한 뒤, 각 단계를 전문 AI Agent가 담당하는 구조화된 Phase를 안내합니다.

## 9개 Phase

모든 워크플로우는 최대 9개의 Phase를 거칩니다. UI가 불필요한 경우 Phase 3처럼 해당하지 않는 Phase는 건너뛸 수 있습니다:

| Phase | 이름 | 진행 내용 |
|-------|------|-----------|
| T | Triage | 요청을 A/B/C 유형으로 분류 |
| 0 | Pre-flight | Git 동기화, 브랜치 생성, 학습 파일 로드 |
| 1 | Discovery | PO 역질문, Working Backwards, PRD, 타당성 평가 |
| 2 | Architecture & Planning | 영향 분석, 설계, API spec, 태스크 분해, 테스트 전략 |
| 3 | UX Design | 화면 설계, SVG 에셋, 디자인 토큰 (조건부 — UI 필요 시에만) |
| 4 | Implementation | BE/FE/DevOps 병렬 구현 |
| 4.5 | Operational Readiness | 배포 전략, 롤백 계획, 운영 준비 상태 점검 |
| 5 | Review & QA | API 계약 검증, Tech Lead + QA + Security 리뷰 |
| 6 | Retrospective | 학습 축적 (전역 학습 + 프로젝트 컨텍스트) |

## 11개 Agent

JARFIS는 11개의 전문 Agent를 조율하며, 각 Agent는 현재 Phase와 프로젝트 요건에 맞을 때만 활성화됩니다:

| Agent | 역할 |
|-------|------|
| **Product Owner (PO)** | 역질문, Working Backwards, PRD 작성 |
| **Architect** | 타당성 평가, 영향 분석, 아키텍처 설계, ADR |
| **Tech Lead** | API spec 검토, 태스크 분해, 코드 리뷰, Retrospective |
| **UX Designer** | 화면 설계, 인터랙션 설계, SVG 에셋 제작, 디자인 토큰 |
| **Backend Engineer** | 백엔드 구현 |
| **Frontend Engineer** | 프론트엔드 구현 |
| **DevOps/SRE** | 인프라 및 CI/CD 구현 |
| **QA Engineer** | 테스트 전략, QA 검증 |
| **Security Engineer** | 사전 보안 분석, 보안 리뷰 |
| **Advocate** | 제안된 솔루션의 장점 옹호, 트레이드오프 방어 |
| **Critic** | 가정에 의문 제기, 리스크와 약점 식별 |

## Human Gate

JARFIS에는 진행 전 명시적 승인이 필요한 3개의 Gate가 있습니다:

| Gate | 시점 | 선택지 |
|------|------|--------|
| Gate 1 | Phase 1 (Discovery) 이후 | 승인 / 수정 / 중단 |
| Gate 2 | Phase 2 & 3 (Architecture + UX) 이후 | 승인 / 수정 / 중단 |
| Gate 3 | Phase 5 (Review & QA) 이후 | 승인 / 수정 후 재검토 / 중단 / 설계 재검토 |

Gate는 매 주요 마일스톤에서 워크플로우의 방향을 여러분이 직접 통제하도록 보장합니다.
