# 快速开始

与AI一起交付软件 — 一条斜杠命令，十一位专业Agent，带Human Gate的结构化Phase。

## 前提条件

开始之前，请确保您已具备：

- **Claude Code**（Anthropic）— JARFIS以斜杠命令工作流的形式原生运行在Claude Code中
- 具有API访问权限的Claude账号
- **Git** >= 2.x（用于Phase 0的分支管理）
- 已克隆或可访问[JARFIS GitHub仓库](https://github.com/sana-lazystar/jarfis)

## 安装

JARFIS通过GitHub仓库的`install.sh`脚本安装。在终端中运行以下命令：

```bash
bash install.sh
```

这将在您的Claude Code环境中注册JARFIS斜杠命令。安装后，以下项目结构可用：

```
your-project/
├── .jarfis/
│   ├── context.md             # 项目特定上下文
│   └── project-profile.md     # 项目概要
├── .jarfis-state.json         # 工作流状态追踪
└── ~/.claude/
    └── jarfis-learnings.md    # 全局学习文件（跨项目）
```

> **注意**：不存在`/jarfis:install`命令。安装仅通过`bash install.sh`完成。

## 运行第一个工作流

在项目目录中打开Claude Code并启动工作流：

```
/jarfis:work Build a user authentication system with JWT tokens
```

JARFIS将对您的请求进行分类（Phase T — Triage），然后引导您完成由专业AI Agent负责的结构化Phase。

## 9个Phase

每个工作流最多经过9个Phase。当某个Phase不适用时可以跳过（例如，不需要UI时跳过Phase 3）：

| Phase | 名称 | 执行内容 |
|-------|------|---------|
| T | Triage | 将请求分类为A/B/C类型 |
| 0 | Pre-flight | Git同步、分支创建、加载学习文件 |
| 1 | Discovery | PO逆向提问、Working Backwards、PRD、可行性评估 |
| 2 | Architecture & Planning | 影响分析、设计、API规范、任务分解、测试策略 |
| 3 | UX Design | 界面设计、SVG资产、设计令牌（条件触发 — 仅在需要UI时） |
| 4 | Implementation | BE/FE/DevOps并行实现 |
| 4.5 | Operational Readiness | 部署策略、回滚计划、运维就绪 |
| 5 | Review & QA | API契约验证、Tech Lead + QA + Security审查 |
| 6 | Retrospective | 学习积累（全局learnings + 项目context） |

## 11个Agent

JARFIS协调11个专业Agent，每个Agent仅在与当前Phase和项目需求相关时才被激活：

| Agent | 角色 |
|-------|------|
| **Product Owner (PO)** | 逆向提问、Working Backwards、PRD起草 |
| **Architect** | 可行性评估、影响分析、架构设计、ADR |
| **Tech Lead** | API规范审查、任务分解、代码审查、Retrospective |
| **UX Designer** | 界面设计、交互设计、SVG资产创建、设计令牌 |
| **Backend Engineer** | 后端实现 |
| **Frontend Engineer** | 前端实现 |
| **DevOps/SRE** | 基础设施和CI/CD实现 |
| **QA Engineer** | 测试策略、QA验证 |
| **Security Engineer** | 预审安全分析、安全审查 |
| **Advocate** | 为提议的解决方案辩护，维护权衡决策 |
| **Critic** | 质疑假设，识别风险和弱点 |

## Human Gate

JARFIS包含3个Gate，在继续之前需要您的明确审批：

| Gate | 时机 | 选项 |
|------|------|------|
| Gate 1 | Phase 1（Discovery）之后 | 批准 / 修改 / 中止 |
| Gate 2 | Phase 2 & 3（Architecture + UX）之后 | 批准 / 修改 / 中止 |
| Gate 3 | Phase 5（Review & QA）之后 | 批准 / 修改后重审 / 中止 / 重新审视设计 |

这些Gate确保您在每个重要里程碑处掌控方向。
