# 架构与概念

JARFIS（Just A Rather Foolish Integration System）通过结构化的Phase流水线协调11个专业AI Agent，交付生产级软件。

## 编排模型

JARFIS以斜杠命令工作流的形式在**Claude Code**内原生运行。它不是独立的CLI工具或外部服务，而是利用Claude Code环境，通过Phase流水线协调11个专业Agent。

## 状态管理

JARFIS在项目根目录的`.jarfis-state.json`中跟踪工作流状态。

```json
{
  "work_name": "auth-system",
  "docs_dir": ".jarfis/works/2026-03-05/auth-system",
  "branch": "jarfis/auth-system",
  "base_branch": "main",
  "current_phase": 4,
  "required_roles": {
    "backend": true,
    "frontend": true,
    "ux": true,
    "devops": false,
    "security": true
  },
  "api_spec_required": true,
  "phases": {
    "T": { "status": "completed" },
    "4": { "status": "in_progress" },
    "4.5": { "status": "pending" }
  }
}
```

主要字段：
- **`current_phase`** — 表示当前活跃Phase的数字或`"done"`
- **`required_roles`** — 控制哪些Agent被激活的布尔标志
- **`phases`** — 各Phase的状态追踪（pending / in_progress / completed / skipped）

## 学习系统

JARFIS通过两级学习系统，随着工作流的积累不断进化。

### 全局学习：`~/.claude/jarfis-learnings.md`
存储在用户主目录，所有项目共享。包含：
- **Agent Hints** — 从Retrospective中积累的行为规则
- **Workflow Patterns** — 经过验证的重复模式

### 项目上下文：`.jarfis/context.md`
注入Agent提示词的项目特定知识：
- 技术栈详情和convention
- 业务领域知识
- 集成约束和现有模式

### 项目概要：`.jarfis/project-profile.md`
由`/jarfis:project-init`生成，由`/jarfis:project-update`更新。

## 设计原则

1. **专业化** — 每个Agent在一个领域拥有深度专业知识
2. **阶段交付** — 结构化Phase确保完整性，不会意外跳过任何步骤
3. **Human Gate** — 重要决策在3个检查点要求明确的用户审批
4. **条件激活** — Agent和Phase仅在相关时激活
5. **有状态** — 工作通过`.jarfis-state.json`跨会话边界持久化
6. **持续学习** — 每次Retrospective通过全局和项目学习改进未来工作流
7. **Claude Code原生** — 无需外部服务，完全在开发环境内运行
