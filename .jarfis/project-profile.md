## Coding Conventions
- **파일 네이밍**: PascalCase.tsx (컴포넌트), kebab-case.ts (유틸/설정), kebab-case.json (데이터), kebab-case.mdx (콘텐츠)
- **컴포넌트 구조**: feature-based co-location (`components/home/`, `components/docs/` 등)
- **Import 규칙**: `@/*` 절대경로 alias (루트 기준), barrel exports 미사용
- **Server Component 기본**: `'use client'` 없음이 기본. 인터랙션이 필수인 경우에만 Client Component

- **다크 모드**: Living Ecosystem 디자인 — 다크 모드 전용 (`#0A1628` 배경). `app/globals.css`의 `@theme` 색상 변수 사용
- **접근성**: WCAG 2.1 AA — skip link, aria-label, aria-labelledby, focus-visible, prefers-reduced-motion
- **타입 안전성**: TypeScript strict mode, `interface`/`type` 명시, `as any` 금지

## Architecture
- **패턴**: SSG (Static Site Generation) — `next build` 시 모든 페이지 정적 HTML 생성, `out/` 디렉토리로 출력
- **i18n 전략**: next-intl App Router 방식. URL prefix routing (`/en/`, `/ko/` 등), `[locale]` 동적 세그먼트, locale별 `generateStaticParams()`, fallback 없음 (각 locale 독립)
- **렌더링**: Server Component 기본. Client Component는 인터랙션 필수 시만 (`'use client'`)
- **MDX 파이프라인**: `lib/mdx.ts`의 `compileMDX` → `next-mdx-remote` → rehype 플러그인 체인 → 커스텀 컴포넌트 매핑
- **SEO**: `lib/seo.ts`의 `generateSeoMetadata()` → Next.js `Metadata` 객체 반환 (OG, Twitter, canonical, hreflang). `components/seo/JsonLd.tsx`로 JSON-LD 구조화 데이터
- **배포**: GitHub Pages (SSG output → `out/` → `gh-pages` 브랜치), `basePath: '/jarfis-website'`

- **lib/docs.ts**: MDX docs 파일 시스템 탐색, slug 파싱, frontmatter 추출, 카테고리별 정렬
- **lib/blog.ts**: MDX blog 파일 시스템 탐색, slug 파싱, frontmatter 추출, 날짜 역순 정렬

- **lib/seo.ts**: 페이지별 SEO 메타데이터 빌더 (canonical URL, OG image resolution, alternate locale links)

## Notes & Caveats
- **SSG basePath**: `basePath: '/jarfis-website'` 설정으로 GitHub Pages 서브경로 배포. 내부 `Link` 및 `Image` 컴포넌트는 자동 처리되나, Markdown 내 상대 링크는 수동으로 basePath 포함 필요
- **Tailwind CSS v4**: `@tailwindcss/postcss` 플러그인 방식 (v3의 `tailwind.config.js` 없음). 모든 디자인 토큰은 `app/globals.css`의 `@theme` 블록에 정의
- **Living Ecosystem**: 다크 모드 전용 디자인 시스템. `--color-neutral-dark: #0A1628`, `--color-surface: #0F1D32`, `--color-primary: #0D9488`, `--color-accent-coral: #FB7185`, `--color-accent-yellow: #FBBF24`
- **Features 페이지 에이전트**: `AgentKey` union type + `AGENT_KEYS` 배열 + `AGENT_AVATAR_STYLES` 객체 패턴. 현재 11개 에이전트 (po, architect, tech_lead, ux, backend, frontend, devops, qa, security, advocate, critic)
- **i18n 342개 키**: en.json 기준. `pnpm i18n:check`로 ko/ja/zh 동기화 검증
- **MDX 커스텀 컴포넌트**: `lib/mdx.ts`의 `compileMDX` components 매핑에 등록된 태그만 MDX 내에서 사용 가능. 기존 마크다운 테이블은 영향 없음
- **TODO/FIXME 없음**: 코드베이스에 TODO/FIXME/HACK 주석 미발견
