## Coding Conventions
- **파일 네이밍**: 컴포넌트 PascalCase.tsx, 유틸리티 kebab-case.ts, 페이지 page.tsx (Next.js convention)
- **컴포넌트 구조**: feature-based co-location (components/blog/, components/docs/ 등), barrel export 미사용 — 직접 파일 import
- **Import 규칙**: `@/` 절대경로 전용, 상대경로 미사용
- **타입 정의**: interface 선호, lib 파일 내 co-located (별도 .types.ts 파일 없음)
- **에러 핸들링**: 최소화 — fs.existsSync() 체크 후 null 반환 패턴
- **스타일링 패턴**: inline `style={{}}` 우선 (CSS 변수 참조) + Tailwind 클래스 (레이아웃/반응형)
- **서버/클라이언트 분리**: 대부분 Server Component, 상호작용 필요 시만 `'use client'` (4개: LanguageSelector, MobileNav, DocsSidebar, TableOfContents)
- **i18n 패턴**: 서버에서 `getTranslations()` → JSX에서 `t('key')`, 클라이언트에서 `useTranslations()`

## Architecture
- **패턴**: Next.js App Router (file-based routing) + Static Site Generation (SSG)
- **렌더링**: 빌드 타임 정적 생성 (`output: 'export'`) — 런타임 서버 없음
- **레이어**: Page (데이터 fetch + layout) → Component (UI) → Lib (데이터 처리/유틸)
- **콘텐츠 파이프라인**: MDX 파일 → gray-matter(frontmatter) → @mdx-js/mdx evaluate() → ReactElement + 커스텀 컴포넌트 주입
- **i18n 전략**: URL 기반 (`/[locale]/`), locale별 MDX 파일, 없으면 en fallback. UI 문자열은 JSON 파일
- **상태 관리**: 외부 라이브러리 없음. useState/useEffect 로컬 상태만 사용 (4개 클라이언트 컴포넌트)
- **인증**: 없음 — 완전 공개 정적 사이트
- **미들웨어**: 없음 (static export와 호환 불가)

## Business Logic Summary
- **lib/docs.ts**: 문서 CRUD — locale별 MDX 파일 탐색, frontmatter 파싱, category×order 정렬, en fallback 지원. sitemap용 네이티브 locale만 반환하는 함수 분리
- **lib/blog.ts**: 블로그 CRUD — locale별 MDX 탐색, pubDate 역순 정렬, en fallback. docs.ts와 동일한 패턴
- **lib/mdx.ts**: MDX 컴파일 엔진 — `evaluate()`로 서버 사이드 컴파일, 19개 커스텀 컴포넌트 + MdxAnchor(내부/외부 링크 자동 분기) 주입
- **lib/paths.ts**: basePath 관리 — GitHub Pages 배포용 경로 prefix. 이중 적용 방지 로직 포함. Next.js Link vs 순수 HTML 태그 구분이 핵심
- **lib/seo.ts**: SEO 메타데이터 팩토리 — canonical URL, hreflang, OpenGraph, Twitter Card, robots 지시자 일괄 생성

## Notes & Caveats
- **TODO/FIXME 없음**: 코드베이스에 기술 부채 마커가 없음
- **basePath 이중 적용 주의**: `withBasePath()`는 순수 HTML 태그(`<a>`, `<img>`, `<link>`)에만 사용. Next.js `<Link>`에서는 basePath가 자동 적용되므로 사용 금지
- **Astro 레거시**: `_src_astro_legacy/` 디렉토리와 `dist/` (Astro 빌드 산출물)가 존재. tsconfig에서 exclude 처리됨. 현재는 Next.js로 마이그레이션 완료
- **inline style 우선**: Tailwind 클래스보다 inline style을 많이 사용하는 패턴. CSS 변수(`var(--color-*)`)를 직접 참조
- **CSP 엄격**: `unsafe-eval` 없이 MDX 컴파일 (서버 사이드 `evaluate()` 사용으로 해결)
- **정적 빌드 제약**: `output: 'export'`로 인해 middleware, API routes, ISR 사용 불가
- **이미지 최적화 비활성**: `images.unoptimized: true` (GitHub Pages 정적 호스팅 제약)
- **HeroSolarSystem**: 11개 행성 궤도 SVG 애니메이션 — GPU 최적화(`transform`, `will-change`), reduced-motion 대응
- **globals.css**: 커스텀 `.prose` 스타일이 MDX 콘텐츠 렌더링에 핵심. 수정 시 모든 문서 페이지에 영향
